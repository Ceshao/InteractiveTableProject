# 识别图与模型映射系统 — 设计文档

> 项目：InteractiveTableProject
> 日期：2026-07-06
> 状态：已确认（用户批准）
> 前置：TUIO 识别链路已打通（commit 388c749）

---

## 一、需求

建立**识别图 ID ↔ 3D 模型**的映射表，通过不同识别图控制不同 3D 模型。

- 本期目标：识别图 id=0 控制场景中 model_A（景泰蓝）的**位置**。
- 物理前提：识别图贴在方块上，方块**常驻桌面不拿走**；标记丢失多为遮挡/抖动。
- 扩展要求：后续新增展品时零代码（只改配置表）；旋转同步、动画、介绍 UI 等留有扩展位。

## 二、架构（四模块分层）

```
[Tuio Session (场景已有)] ─ Tuio11Dispatcher 的 Add/Update/Remove 事件
         │
         ▼
① MarkerModelManager（输入/调度层，场景单例 MonoBehaviour）
   监听 TUIO Object 事件 → 按 SymbolID 查配置表 → 实例化/驱动对应控制器
         │ 查表
         ▼
② MarkerContentLibrary（配置层，ScriptableObject 资产）
   映射表：SymbolID → 模型 Prefab + 行为参数
         │ 实例化/复用
         ▼
③ TrackedModelController（表现层，挂在模型实例上）
   持有 Tuio11Object 引用 → 每帧坐标换算 → 平滑移动模型
         │ 换算委托给
         ▼
④ TuioWorldMapper（坐标映射工具，独立静态/小组件）
   TUIO 归一化坐标(0~1, y向下) → 正交相机可视平面的世界坐标
```

职责边界：①只管事件分发、②只管数据配置、③只管模型表现、④只管坐标数学。
加旋转同步改③；加新展品只改②的资产；改投影方式（如 XZ 地面）只改④。

## 三、配置表结构

`MarkerContentLibrary.asset` 的条目（List）：

| 字段 | 类型 | 说明 | 首条数据 |
|---|---|---|---|
| symbolId | int | 识别图 ID | 0 |
| modelPrefab | GameObject | 模型 Prefab | Model_A.prefab |
| lostPolicy | enum | 丢失策略：KeepInPlace / Hide / HideAfterDelay | KeepInPlace |
| smoothTime | float | 位置平滑时间（秒） | 0.1 |

- 编辑器 OnValidate 检查重复 symbolId 并告警。
- 前置动作：把场景中的 model_A（含已调好的 scale 0.03、rotation X-90）固化为
  `Assets/Prefabs/Model_A.prefab`，场景中原对象移除，由管理器按需实例化。

## 四、行为定义

- **坐标映射**：`世界坐标 = 主相机.ViewportToWorldPoint(x, 1-y)`，Z 保持模型原深度。
  桌面边缘对应屏幕边缘（正交相机）。
- **位置平滑**：SmoothDamp，时间常数 smoothTime，防识别抖动。
- **Remove（丢失）**：默认 KeepInPlace——模型停在最后位置，重新识别到继续跟随。
- **未知 ID**：日志提示一次，忽略，不报错。
- **同 ID 重复放置**：第一个生效，后续忽略并告警。
- **本期不做**（已留扩展位）：旋转同步、出场/退场动画、介绍 UI 面板。


## 六、测试方式（真实环境端到端）

1. 启动 reacTIVision（内置摄像头，DirectShow index 0）。
2. Unity 进入 Play 模式（Tuio Session：UDP / 127.0.0.1 / 3333 / Tuio11）。
3. 将打印的识别图对准摄像头并移动，验证：
   - ① model_A 出现并跟随移动；
   - ② 移动平滑无抖动；
   - ③ 遮挡识别图再露出，模型停在原地后恢复跟随。
4. 识别图 ID 以 reacTIVision 窗口实时显示为准；若实际非 0，仅修改配置表该行数字。

注意：识别图需清晰、平整、不反光；测试时光线均匀。
