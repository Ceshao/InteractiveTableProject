# 桌面互动应用 — 技术方案与模拟测试指南

> 项目：InteractiveTableProject
> 技术范式：Tangible Tabletop（有形物件桌面交互）/ Reactable
> 更新日期：2026-07-02

---

## 一、需求概述

做一个**桌面互动应用**：在桌面上放置 3D 打印方块（底部粘贴 fiducial 标记），
通过红外相机识别方块的身份、位置与角度，运行 Unity 程序在大屏上实现互动展示
（如放上某个方块即调出对应展品的 3D / 介绍）。

### 现有条件
- reacTIVision-1.5.1-win64 运行程序（已准备）
- Unity 工程已导入 `TuioUnityClient`（https://github.com/InteractiveScapeGmbH/TuioUnityClient.git）
- Unity 版本：2022.3.8f1 LTS
- 当前仅有一台笔记本，暂无现场调试环境

### 已确认的物理方案
| 项目 | 结论 |
| --- | --- |
| 桌面感应结构 | **背投式**（桌面为磨砂扩散屏，桌内投影 + 桌下红外相机从下往上拍标记） |
| 内容展示屏 | 内容显示在**墙面 / 独立大屏**，桌面主要作为感应操作台 |
| 核心玩法 | 尚在构思，**先跑通技术链路（MVP）** |

## 二、技术可行性结论

**整体成熟可行。** 这是经典的 Tangible Tabletop 范式，reacTIVision + TuioUnityClient
是该范式最标准的开源组合，博物馆 / 展厅已大量验证。

**关键风险已排除**：背投式结构是 reacTIVision 官方标准结构，红外相机能干净地拍到方块
底面标记，不存在「红外透不过 LCD」的问题。技术上可放心推进。

## 三、整体架构与数据流

```
[3D打印方块 + 底部粘贴 amoeba/fiducial 标记]
            │  放在桌面上
            ▼
[红外照明 + 红外相机]  ── 拍到标记的黑白图案
            │  视频帧
            ▼
[reacTIVision-1.5.1  (独立 exe)]
   · 识别 SymbolID（标记编号）
   · 计算 位置(x,y 归一化 0~1)、角度、cursor(手指触摸)
            │  TUIO 1.1 协议，UDP :3333（OSC 封装）
            ▼
[Unity + TuioUnityClient]
   · TuioSessionBehaviour  ← 接收 UDP
   · Tuio11 分发器          ← 解析 Object/Cursor 的 add/update/remove
   · 业务脚本               ← SymbolID → 生成对应 Prefab / 触发内容
            │  渲染
            ▼
[墙面 / 独立大屏]  展示互动内容
```

**关键点**：reacTIVision 是独立运行的 .exe，不在 Unity 内部。它把视觉识别结果通过
UDP 3333 发出，Unity 仅作为 TUIO 客户端接收。两者靠 localhost 网络解耦（生产者—消费者）。

## 四、各环节技术要点

### ① 标记（Marker）
- reacTIVision 使用 **amoeba 类标记**（自带 `default.trees` 可识别约 216 个 ID）。
- 3D 打印方块把标记贴在**接触桌面的底面**，尺寸建议边长 ≥ 4~5cm，标记要清晰、平整、无反光。

### ② reacTIVision 配置（`reacTIVision.xml`）
- 输出协议默认 **TUIO 1.1 / UDP / `host=127.0.0.1 port=3333`**，与 TuioUnityClient 默认一致，基本零配置。
- 需做一次**相机标定**（内置 calibration 模式 + 标定板），校正广角镜头畸变。

### ③ TuioUnityClient（已导入 v2.0.4）
核心组件：
- `TuioSessionBehaviour`：连接管理，可配置 `TuioVersion` / `ConnectionType` / `UdpPort` / `IpAddress`
- `Tuio11` 系列：`Tuio11Behaviour`、`Tuio11ObjectTransform`、`Tuio11CursorTransform`、`Tuio11Visualizer` 等

业务代码要订阅的事件：
- **Object add/update/remove** → 用 `SymbolId` 查表决定显示内容，用 `Position(0~1)`、`Angle` 摆放 / 旋转。
- **Cursor**（手指触摸，若桌面支持）→ 点选、拖拽。

**坐标映射**：TUIO 给的是 0~1 归一化坐标，需转成 Unity 屏幕 / 世界坐标，并处理**镜像翻转**
（相机在桌下拍，X 或 Y 常需翻转）。这是实际调试最花时间的一步，建议做成 Inspector 可调开关。

### ④ 大屏展示
Unity 可多显示器输出（桌面屏做操作层 + 墙面屏做展示层），或直接全屏到墙面大屏。

## 五、⚠️ 关键一致性提醒：TUIO 版本必须匹配

- **reacTIVision 1.5.1 实际发送的是 TUIO 1.1**（`2Dobj` / `2Dcur`）。
- 当前工程导入的是 **TUIO 2.0 Example Scene** —— 版本不匹配，reacTIVision 的消息一条都收不到。
- **正确做法**：使用 **TUIO 1.1** 链路，本番也必须让
  **reacTIVision(TUIO 1.1) ↔ Unity(TuioVersion = Tuio11)** 版本一致。

## 七、模拟测试方案（仅需一台笔记本，离线完成）

无现场环境也能完整验证「标记识别 → TUIO → Unity 响应」链路。除物理外壳外，其余问题可提前排除。

### 方式 2️⃣（推荐·最接近真实）：reacTIVision + 笔记本内置摄像头 + 打印标记
直接用已有的 reacTIVision，现场用的是同一套识别软件、吐出同样的 TUIO 1.1 数据，等于本番链路预演。

1. 打印 reacTIVision 自带的 **amoeba 标记 PDF**（在 `reacTIVision-1.5.1` 文件夹的 `symbols` / 手册里），
   剪下来贴在厚纸片或 3D 方块上（直接拿纸片对着摄像头也行）。
2. 运行 `reacTIVision.exe`，输入源设为**笔记本内置摄像头**。预览窗里标记被框住并显示 ID 即识别成功。
3. reacTIVision 自动向 **UDP `127.0.0.1:3333` 发送 TUIO 1.1**。
4. 按下方 Unity 通用设置配置接收 → 在摄像头前移动标记，Unity 内物体跟随移动 / 旋转。

> 跑通后，现场只剩「装到背投桌 + 镜头标定 + 坐标翻转对齐」等机械工作。

### 方式 1️⃣（最快·纯软件·无需摄像头）：TUIO Simulator
只想先看「Unity 能否正确接收·解析·显示」时用。

1. 运行官方 **TUIO Simulator**（Java 程序 `TUIO_Simulator.jar`，从 tuio.org 下载，需 Java 运行时）。
2. 窗口内用鼠标拖动虚拟 fiducial 方块（带 ID）和手指 cursor → 向 **UDP 3333 发送 TUIO 1.1**。
3. 按下方 Unity 通用设置配置接收即可看到响应。

### Unity 侧通用设置（两种方式相同）
1. **Package Manager → TUIO Client → Samples → 导入「TUIO 1.1」**（不是 2.0）。
2. 打开 `TUIO 1.1 Example Scene`。
3. 选中挂 `TuioSessionBehaviour` 的物体，在 Inspector 设置：
   - **TuioVersion = Tuio11**
   - **ConnectionType = UDP**
   - **UdpPort = 3333**
   - **Ip Address = `127.0.0.1`**（默认 `10.0.0.20` 在本机接收时可能收不到，务必改掉）
4. ▶ Play → 在 Simulator / 摄像头前移动标记，画面上的物体跟随位置与角度。
