# X 镜像热键切换 — 设计

日期:2026-07-14
背景:现场 reacTIVision.xml 的 `<calibration ... invert="x" />` 使 X 坐标镜像,导致我们的
程序左右移动相反。已部署的旧程序依赖该配置,不能删,由我们的程序做兼容。

## 方案

在 TUIO 数据入口做一次 X 镜像抵消,F2 热键切换,PlayerPrefs 记忆:

1. **开关**:`MarkerInputMirror`(Core,静态类)。`InvertX` 从 PlayerPrefs
   (`TangibleTable.InvertX`,默认关)初始化;`Toggle()` 翻转并立即落盘。
2. **修正点**:`TuioMarkerSource.TuioTrackedMarker.Position` 读取时
   `x → InvertX ? 1-x : x`。在入口修正,桌面光标、按钮命中、UDP 转发给投影的坐标
   一次性全部一致,Projection 端零改动。
3. **热键**:`MirrorToggleHotkey`(TableUI 场景)。F2 切换;切换后提示 3 秒;
   镜像开启期间左下角常驻"X 镜像:开"小标记,现场一眼可辨。
   F1 已被投影诊断面板占用,故用 F2。
4. **场景**:组件已加入 TableUI.unity;`PocSceneBuilder` 同步更新,重建场景不丢。

## 现场操作

桌面屏窗口获得焦点(点一下桌面屏)后按 F2。状态存 PlayerPrefs,重启自动恢复,
只需设置一次。

## 边界

- 角度不镜像:reacTIVision 的 invert 只翻坐标,旋转角不受影响。若现场发现旋转方向
  也反了,再把角度镜像挂到同一开关上。
- Angle 修正预留:如需扩展,改 `TuioTrackedMarker.Angle` 即可,开关复用 `InvertX`。

## 验证

- 编辑器 Play + tuio_sender.py 伪造数据:按 F2 前后光标 X 对称翻转,按钮命中区正确。
- 重启 Play:镜像状态保持。
